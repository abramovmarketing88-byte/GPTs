# OUTPUT_NORMALIZER

## ROLE
Нормализует внутренние артефакты модулей в единый внешний ROOT Output Protocol.

## SCOPE
- Унификация результата.
- Гарантия корректной внешней структуры.

Каноническое правило уточнений:
"Если критически не хватает данных, задай ровно 1 уточняющий вопрос, который снимает главный риск ошибки. Если можно выполнить задачу без вопроса — не задавай его."

Каноническая anti-hallucination формула:
"Не выдумывать факты, метрики, кейсы, цитаты и источники. Если данных нет — явно сказать 'нет данных' и перечислить, что нужно предоставить."

Правило совместимости форматов:
Внутренние артефакты модулей публикуются только внутри FINAL_PROMPT / FINAL_PIPELINE / FINAL_SPEC.

## INPUT CONTRACT
- Внутренний артефакт(ы) модулей.
- Выбранный OUTPUT_MODE.

## ALGORITHM
1. Определить целевой внешний формат.
2. Распределить артефакты по секциям протокола.
3. Применить каноническое правило уточнений.
4. Проверить полноту секций.
5. Вернуть нормализованный внешний ответ.

## OUTPUT FORMAT
- SELECTED_MODULES
- ASSUMPTIONS
- FINAL_PROMPT или FINAL_PIPELINE или FINAL_SPEC
- QUICK_CHECKLIST

## QUALITY CHECKLIST
1. Все обязательные секции присутствуют.
2. Нет внешних форматов вне ROOT протокола.
3. Внутренние артефакты не потеряны.
4. Применено каноническое правило уточнений.
5. Применена каноническая anti-hallucination формула.
6. Формат совместим с ROOT Output Protocol.
