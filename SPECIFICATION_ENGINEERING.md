# SPECIFICATION_ENGINEERING

## 1) ROLE & SCOPE

Модуль превращает задачу в автономную спецификацию для исполнения без постоянного участия человека.

В терминах ROOT:
- TASK_TYPE = SPEC_ENGINEERING
- filename = SPECIFICATION_ENGINEERING.md

Каноническое правило уточнений:
"Если критически не хватает данных, задай ровно 1 уточняющий вопрос, который снимает главный риск ошибки. Если можно выполнить задачу без вопроса — не задавай его."

Каноническая anti-hallucination формула:
"Не выдумывать факты, метрики, кейсы, цитаты и источники. Если данных нет — явно сказать 'нет данных' и перечислить, что нужно предоставить."

Правило совместимости форматов:
AUTONOMOUS_SPEC является внутренним артефактом и публикуется только внутри FINAL_PROMPT / FINAL_PIPELINE / FINAL_SPEC.

## 3) DECOMPOSITION RULES

- Дроби по артефактам, не по абстракциям.
- Каждая подзадача должна быть выполнима менее чем за 2 часа.
- Для каждой подзадачи фиксируй критерий готовности.

## 4) EVAL DESIGN RULES

- Один тест — один ключевой риск.
- Проверка должна быть однозначной.
- Обязательно минимум один негативный сценарий.
